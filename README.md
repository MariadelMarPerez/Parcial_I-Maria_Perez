# Maquetación Web - ExpressVPN

Este proyecto consiste en elaboracion de la landing page de ExpressVPN. El objetivo es replicar el diseño visual proporcionado utilizando buenas prácticas de desarrollo web en html y css, estructura semántica y control de versiones git.

* **Maria del Mar Perez Montenegro**

## Estructura del Proyecto
Para mantener el orden y cumplir con los requisitos del parcial, el proyecto está organizado de la siguiente manera:

* **index.html**: Contiene toda la estructura y el contenido de la página.
* **style.css**: Contiene las reglas de diseño, colores y la organización visual.
* **src/**: Carpeta que almacena todos los recursos gráficos (imágenes y logos) utilizados asi:
    * `foto1.png`: Imagen principal de la sección.
    * `foto2.png`, `foto3.png`, `foto12.png`: Imágenes de las secciones de beneficios, logo y marcas.

## Secciones de la landing
La página se divide en bloques claros y fáciles de identificar:

1. **Encabezado (Header):** Incluye el logo de ExpressVPN y un selector de idioma. Se diseñó para que sea lo primero que se identifique.
2. **Sección Principal (Hero):** Presenta el título principal "The VPN that just works" y una imagen representativa a la derecha.
3. **Sección de Beneficios (Why ExpressVPN?):** Muestra tres características clave. Se organizaron de forma alterna (imagen-texto / texto-imagen) para que la lectura sea clara.
4. **Sección de Prensa:**: franja con logos de marcas (CNET, BBC, etc.)
5. **Pie de Página (Footer):** Espacio con los créditos del autor, mención al parcial y enlaces a políticas de privacidad.

## Elementos y Tecnologías Utilizadas
Para este trabajo se utilizaron las siguientes herramientas:

* **HTML Semántico:** Uso de etiquetas como `<header>`, `<main>`, `<section>`, `<article>` y `<footer>` para que el código quede bien estructurado.
* **CSS con Flexbox:** Se usó para alinear los elementos (como poner textos al lado de imágenes) de forma flexible y sencilla.
* **Variables CSS:** Se definieron los colores  (verde, crema y azul oscuro) para que el diseño sea consistente con la plantilla.
* **Git / GitHub:** El proyecto se trabajó usando ramas (`maria-perez`) y se subió con mensajes de commit descriptivos para registrar el avance.
